Microsoft Copilot (Microsoft 365): Generate documents using forms in SharePoint with Power Automate

We are introducing a new Power Automate action that enables automated document generation using document generation (DocGen) forms in SharePoint. Generate document from form (Preview) allows customers to create documents from a template by mapping data inputs to predefined template fields. Documents are generated as part of an automated flow and stored in SharePoint. This capability supports scenarios such as creating contracts, invoices, letters, or reports using structured templates and form-driven inputs. The action works by mapping input data to template fields and returns the generated document along with file metadata such as location and status.

SharePoint: Power Automate trigger and action for forms in SharePoint

We are introducing new Power Automate integration capabilities for forms in SharePoint, enabling customers to create automated workflows based on responses collected using list forms, file upload forms, and document generation forms. With this update, a new trigger and action are now available in Power Automate: Trigger: When a form is submitted This operation triggers a flow when a list form, document generation form, or file upload form is submitted. Customers can use this trigger to initiate automated workflows such as approvals, notifications, or routing based on form submission events. Action: Get form metadata This action retrieves the metadata of a form, which includes the form ID, title, link, form type, output format, and the form questions used to collect information. This enables Power Automate flows to dynamically read form schema before executing downstream automation logic. Disclaimer: Document generation forms are part of AI in SharePoint Public Preview. Access to DocGen Forms capabilities may be limited to customers enrolled in the preview program. List forms and file upload forms are not part of the AI in SharePoint preview.
