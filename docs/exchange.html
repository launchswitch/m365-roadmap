Exchange: Exchange Online Moderation approval requests via Actionable Messages

The Exchange Online Moderation feature gives message moderators the ability to Approve or Reject sending a moderated message to its intended recipients via a special email message form that includes Approve and Reject buttons in the Outlook Desktop or Outlook on the web toolbar. However, the ability to approve or reject a moderated message currently isn't supported on Outlook for iOS, Outlook for Android, or Outlook for Mac. With this update to use Actionable Messages for moderation approval messages, moderators with mailboxes hosted in Exchange Online will now be able to approve or reject moderated messages via all the Microsoft email clients: from Outlook for Windows, Outlook for Mac, Outlook for iOS and Outlook for Android.

Status: Rolling out | GA Date: March CY2026 | Roadmap Status: Include this month

Products: Exchange

Microsoft 365: SMTP onboarding to App Role Based Access Control

We're simplifying how organizations grant applications permission to send email on behalf of mailboxes. Today, customers must manually assign permissions to each individual mailbox using PowerShell, which is time-consuming and inefficient. With this new capability, admins can assign the SMTP.SendAsApp role to an app through App Role-Based Access Control (RBAC), enabling group-based or scoped access to mailboxes. This eliminates the need for per-mailbox configuration and streamlines onboarding for SMTP clients using OAuth. It’s a scalable, secure, and modern approach to managing mailbox access for applications.

Exchange: New Message Trace in Exchange Online

Admins can access the new Message Trace and its capabilities by default when navigating to the Exchange admin center > Mail flow > Message Trace. Admins can also leverage the new Message Trace PowerShell cmdlets Get-MessageTraceV2 and Get-MessageTraceDetailV2.
